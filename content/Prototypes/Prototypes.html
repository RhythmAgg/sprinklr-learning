Prototypes

In JavaScript, objects have a special hidden property [[Prototype]] (as named in the specification), that is either null or references another object. That object is called “a prototype”. When we read a property from object, and it’s missing, JavaScript automatically takes it from the prototype. In programming, this is called “prototypal inheritance”.

let animal = {
  eats: true
};
let rabbit = {
  jumps: true
};

rabbit.__proto__ = animal; // sets rabbit.[[Prototype]] = animal
// OR
let rabbit = {
  jumps: true,
  __proto__: animal
};
let user = {
  name: "John",
  surname: "Smith",

  set fullName(value) {
    [this.name, this.surname] = value.split(" ");
  },

  get fullName() {
    return `${this.name} ${this.surname}`;
  }
};

let admin = {
  __proto__: user,
  isAdmin: true
};

alert(admin.fullName); // John Smith (*)

// setter triggers!
admin.fullName = "Alice Cooper"; // (**)

alert(admin.fullName); // Alice Cooper, state of admin modified
alert(user.fullName); // John Smith, state of user protected

NOTE: this refers to the object before the dot ALWAYS, so it changes the admin state when

admin.fullName = “Alice Cooper” calls the set function

The modern methods to get/set a prototype are:

  • Object.getPrototypeOf(obj) – returns the [[Prototype]] of obj.
  • Object.setPrototypeOf(obj, proto) – sets the [[Prototype]] of obj to proto.
let animal = {
  eats: true
};

// create a new object with animal as a prototype
let rabbit = Object.create(animal); // same as {__proto__: animal}
// we can also pass descriptor
let rabbit = Object.create(animal, {
  jumps: {
    value: true
  }
});

alert(rabbit.eats); // true

alert(Object.getPrototypeOf(rabbit) === animal); // true

Object.setPrototypeOf(rabbit, {}); // change the prototype of rabbit to {}
